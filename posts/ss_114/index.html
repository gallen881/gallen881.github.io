各校系特選過程

國立臺灣大學 資訊工程學系（資安組）

書面審查

臺大資工沒有一般組的特殊選才，只有資安組。雖然簡章上明確地寫了需要資安經歷，但基於試試看的想法還是報了。最後理所當然地一階就被刷掉。

特別需要注意的是臺大的特殊選才是由學校集體報名，不像其他學校是自己報名。有些科系似乎還會限制同校名額？（資工沒有）所以要注意校內消息。

國立清華大學 資訊工程學系Ａ組

書面審查

順利地通過進入複試，不過分數是擦邊，最低標準是 85.5 分。清大資工特別喜歡競程的同學，大部分做專案的分數就落在 90 以下，競程的幾乎都 90 以上。很多科展專案很強的甚至沒通過書面審查。

面試

共分為兩間進行面試，第一間就像是一般的面試。自我介紹五分鐘，教授問答五分鐘，總共十分鐘。第二間則是問演算法問題，時間也是十分鐘。

一開始拿名牌後在休息室等，會有人來叫名字並帶你到面試的教室。

最後備取 13，正取幾乎都是競程的。

國立陽明交通大學 資訊工程學系

交大的特選介面很好看，但是不知道為什麼資工系要特別做一個資工系的上傳介面，因此報名的時候分了兩邊同時填寫。

我拿到的分數應該算中間，記得有電神拿到書審 99 分。

面試

項目 時間（分鐘）
自我介紹 1
教授問答 6
總計 7

報到後在休息室等，會有人來叫名字並帶你到面試的教室。

至於面試時間這間是最嚴格的，會有人在外面以手機計時，時間到會直接開門請你離場，不管現在講到哪。這間的教授也是最嚴肅的，問的問題也很深，會特別刁鑽。另外這間是不收資料的，我有待資料過去問教授是否願意看，教授表示聽我說就好。

在我自我介紹完後教授問我書審資料是不是沒有填好，我覺得很奇怪。交大資工特別做了分離學校的系統上傳資料，但是我很確定兩邊系統都有填寫。回答都有填寫後就進入問答。

問答幾乎都是中間的教授問問題，右邊的教授完全沒說話。過程教授幾乎都在問我知不知道 Hidden Markov Model，我照我知道的回答教授，教授卻一直說很奇怪😥。

國立成功大學 資訊工程學系－乙組

成大資工特選分為甲乙組，甲組是針對競程的，乙組是針對實作的。以資工的特選來說成大資工乙組大概是對做專案的同學最友善的。

書面審查

成大似乎比較在意上機能力，因此書面審查幾乎有報名就有通過。

上機考

每次上機考的內容會不一樣。

這次上機考有用到的軟體工具大致如下：

  • Ubuntu
  • Visual Studio Code
  • GNU C Compiler
  • Python
  • Docker
  • Zenoh
  • ROS2
  • Rvis2
  • Gazebo
  • Navigation 2

題目共分兩大題，第一大題有四小題，第二大題有兩小題。

試前測試

考試前一天他們有寄信過來，可以在指定的時間（兩小時）登入 GitLab 進行測試。這個測試可以先熟悉牠們的解題流程，才不會考試時不會用他們的 Judge。

上機過程

早上八點多就要到，考試在他們的電腦教室進行。考試開始前有一段時間讓你在電腦登入各種帳號，例如 Google、ChatGPT 等。沒錯他可以用 ChatGPT，也可以在考試的時候使用網路查資料。但是禁止使用自己的可通訊電子產品，也不能討論。因此登入完帳號後就要求將手機等產品收入背包放到旁邊。雖然可以用網路查資料，但是不能安裝任何軟體，包括 VS Code 插件等。

接著助教公布題目後有稍微講解題目，如果你超強可以不理它直接開始做。

在題目中有數個 Bug，你必須找到並且修好他，並且照題目的敘述從程式取得資料，最後上傳到他們的 GitLab。他們使用 GitLab CI 作為 Judge，當你 Push 檔案上去後就會開始自動評分，可以在 CI 的 Log 看到自己答題的正確與否。然後開始前的一小時第一題的 Judge 似乎都是壞的，沒有辦法評分💀。

一小時後助教視情況提供了第一題的提示，之後也陸續提供提示。

我在上午解出第一小題後就一直卡在第二小題，然後上午就結束了。

得說他們的環境不是那麼可靠，下午我請他們幫我重裝 Container，原本打不開的程式就打開了（我上午以為程式打不開是我們要解決的 Bug 🤡）。

中午的午餐時間，因為大家都被要求不能討論，所以特別的安靜。（但結果還是看到有人在吃午餐的教室外講話）

最後我大概只解出一到兩題，備取六。

教授面談

當天下午考完試後還有一個能夠和教授聊天的機會，原本以為是面試，但後來的信件有解釋不會計分。解釋內容如下：

  1. 此環節的主要目的是讓系上教授向考生介紹本系課程內容與系上的學習環境，同時也想了解考生的考試情況作為來年考試出題的依據。此外，考生可以藉由此場合詢問教授系上的狀況，以期對本系有更多的了解。
  2. 此面談的部分採自由參加制，不會影響考生的考試分數。本次特殊選才考試分數，將依據考試簡章描述，由書面審查與實際的上機考試成績所決定。

國立中央大學 資訊工程學系

中央在特選裡面算是時程特別早的，需要注意。

這次的兩位正取都是旺宏科學獎金牌，2022 年的正取 1 也是旺宏科學獎金牌。我認為中央較喜歡專案仔，旺宏金牌或許很吃香，想去中央資工的可以試試。

中央資工的面試順序很特別，是照書面審查的成績排的，愈高分愈前面。有些人會被這個影響心態，不要覺得第一個面試贏麻，也不要覺得最後面試輸光。面試規定要用簡報，而且自我介紹必須包含在校成績、特殊才能及傑出表現……等。我在校成績的部分是放了高中數學和資訊課的排名、各學期的排名、學測第一次模擬考成績，傑出表現就放一些競賽。

國立陽明交通大學 百川學士學位學程

書審資料

這是所有遇過最特別的：

  1. 手寫自傳
    規定要用手寫，十分的奇妙，因此有些人會在視覺效果下功夫。我自己是簡單的拿有格線的紙寫完後掃描。

  2. 學習計畫
    要求的很詳細：

    請參照本學程修課規定，由本學程規定的專業核心課程中擇一為入學後預定修習之專業核心課程，並就此專業核心課程與其他修課規定擬訂修課藍圖，此藍圖應就個人學習經歷述明規劃之因由及可行性，限A4三頁內(三面)。學程修課規定與專業核心課程詳參https://aretehp.nycu.edu.tw/或簡章附錄。

    那時候研究課程好久才寫出來。

筆試

這又是很神奇的點，百川除了面試還有筆試！筆試在早上考，通常會有國文寫作、英文題、數學應用題。考古題可以在百川的網站找到，不過個人認為考古題幫助不大。

心得

特殊選才不容易，如果你不保證自己穩上，就還要同時準備學測。像我自己就是學測前一天才確定備取上。高三的課業和學測的複習已經夠忙了，再加一個備審資料的製作和面試練習就要蠟燭三頭燒。當別人再捲學測，而自己沒有再讀書而是一直用電腦做備審，這還是會有些壓力的。想要找他人幫忙，但特殊選才人數少，看過很多同學都是被學校放生的。我自己也是要主動去找老師或社群去了解。每個管道都各有自己的難處，希望這文章能夠幫到要走特殊選材的人。
